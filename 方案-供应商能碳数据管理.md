> 版本：V1.0 | 日期：2026-06-07

## 一、功能概述

供应商能碳数据管理是本系统的重要扩展模块，核心原则：

**供应商只提交原始能耗统计数据，碳评价由系统统一计算。**

实现：
1. **供应商自助门户** — 每个供应商拥有独立账号，登录后可提交原始能耗数据
2. **多渠道数据采集** — Excel 上传（无系统的供应商）+ API 对接（有系统的供应商）+ Odoo/ERP 同步
3. **供应商生命周期管理** — 从列表初始化、账号生成、数据采集到碳评分计算

### 角色与职责边界

| 角色 | 做的事情 | 不做的事情 |
|------|----------|-----------|
| **供应商** | 提交原始能耗统计数据（用电量、天然气...） | ❌ 不评价、不自算碳排、不评级 |
| **本系统** | 统一排放因子计算碳排 → 自动评分 | ✅ 评分标准公开透明 |
| **管理员** | 审核数据、管理供应商、配置因子 | ❌ 不手动评分 |

## 三、供应商账号体系

### 3.1 账号生成规则

每创建一个供应商，系统**自动生成**一个独立账号：

| 字段 | 规则 | 示例 |
|------|------|------|
| **用户名** | `gp_{供应商编码（小写无空格）}` | `gp_shanghai_steel` |
| **初始密码** | `GP@{6位随机大写字母}{2位数字}` | `GP@AXFKLQ37` |
| **强制改密** | 首次登录强制修改密码 | — |
| **邮箱（可选）** | 供应商联系人邮箱 | shanghai@steel.com |
| **状态** | active / disabled | active |

**用户名规则备选方案**：
```
方案A：gp_{拼音缩写}     gp_shsteel
方案B：sup_{数字ID}     sup_10001    ← 推荐
方案C：gp_{公司名简拼}   gp_shlc
```

**密码规则**（自动生成，符合安全规范）：
```
长度12位：大写+小写+数字+特殊字符
例：Gp#8kLx@3qWz
```

### 3.2 供应商登录权限

供应商登录后**只能访问自己的数据页面**，不能看到主系统的任何内容：

## 四、供应商数据采集

### 4.1 方式一：Excel 上传（第一阶段唯一方式）

> 无论大供应商还是有系统的供应商，**第一阶段统一采用 Excel 上传**。  
> 页面会提供 API 接口文档下载，供有开发能力的供应商提前了解。  
> 有任何技术问题可联系：**fengliang@ziwi.cn**

#### 流程

```
系统管理员创建供应商 → 自动生成账号
       ↓
供应商收到用户名密码
       ↓
供应商登录 http://<服务器IP>/gys/
       ↓
下载 Excel 模板 → 填写能碳数据 → 上传
       ↓
系统自动校验 → 写入数据库 → 更新碳评分
       ↓
主系统管理员在"供应商数据看板"查看
```

#### Excel 模板（供应商专用）

供应商只需填写 **原始能耗统计数据**，碳排放和评分由系统自动计算。

| 字段 | 说明 | 类型 | 必填 | 示例 |
|------|------|------|------|------|
| 报告月份 | 数据所属月份 | YYYY-MM | ✅ | 2026-06 |
| 产品产量(件) | 当月产出（用于强度计算） | 整数 | ✅ | 15000 |
| 外购电力(kWh) | 当月用电量 | 数值 | ✅ | 125000 |
| 天然气(m³) | 当月天然气用量 | 数值 | 选填 | 5000 |
| 蒸汽(吨) | 当月蒸汽用量 | 数值 | 选填 | 200 |
| 柴油(升) | 当月柴油用量 | 数值 | 选填 | 3000 |
| 汽油(升) | 当月汽油用量 | 数值 | 选填 | 1000 |
| 备注 | 说明 | 文本 | 选填 | |

> **系统自动计算**：碳排放量 = 各能源消耗量 × 对应排放因子（系统统一配置）
> **系统自动评分**：基于碳排放强度（单位产值排放）计算碳评分
> **供应商无需**：自行计算碳排放、评分或评价

#### 前端页面：供应商数据看板（管理员端）

在主系统「供应链碳管理」页面新增 Tab：

```
供应链碳管理
├─ [供应商总览]  ← 现有的供应商列表
└─ [供应商数据]  ← 新增 Tab
   ├─ 供应商选择下拉 → 查看该供应商各月数据
   ├─ 数据提交状态（已提交/未提交/逾期）
   ├─ 碳排趋势折线图（近12个月）
   └─ 数据导出（Excel）
```

### 4.2 方式二：API 对接（第二阶段启用，页面预留文档）

## 五、数据库设计

### 5.1 供应商主表（扩展已有 `suppliers` → 改用 DB 存储）

```sql
CREATE TABLE suppliers (
    id SERIAL PRIMARY KEY,
    supplier_code VARCHAR(50) UNIQUE NOT NULL,  -- 编码
    supplier_name VARCHAR(200) NOT NULL,         -- 名称
    contact_person VARCHAR(100),                 -- 联系人
    contact_email VARCHAR(200),                  -- 邮箱
    contact_phone VARCHAR(50),                   -- 电话
    carbon_score REAL DEFAULT 0,                 -- 碳评分（自动计算）
    carbon_level VARCHAR(10),                    -- A/B/C/D（自动计算）
    annual_co2_ton REAL DEFAULT 0,               -- 年度碳排放
    status VARCHAR(20) DEFAULT 'active',         -- active/disabled
    source VARCHAR(20) DEFAULT 'manual',         -- manual/odoo/erp/api
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 5.2 供应商账号表

```sql
CREATE TABLE supplier_users (
    id SERIAL PRIMARY KEY,
    supplier_id INTEGER NOT NULL REFERENCES suppliers(id),
    username VARCHAR(100) UNIQUE NOT NULL,       -- 登录用户名
    password_hash VARCHAR(128) NOT NULL,          -- bcrypt 哈希
    api_key VARCHAR(64) UNIQUE,                   -- API Key
    must_change_password INTEGER DEFAULT 1,       -- 首次登录强制改密
    last_login_at TIMESTAMP,
    is_active INTEGER DEFAULT 1,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 供应商 API 对接信息
CREATE TABLE supplier_api_info (
    id SERIAL PRIMARY KEY,
    supplier_id INTEGER UNIQUE REFERENCES suppliers(id),
    api_key VARCHAR(64) UNIQUE,                   -- 对接 API Key
    api_enabled INTEGER DEFAULT 0,                -- 是否启用API（第一阶段=0）
    api_doc_downloaded INTEGER DEFAULT 0,         -- 是否下载过API文档
    contact_email VARCHAR(200),                    -- 对接人邮箱
    developer_contact VARCHAR(100),                -- 开发负责人
    notes TEXT,                                    -- 备注
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 5.3 供应商能碳数据表

```sql
CREATE TABLE supplier_carbon_data (
    id BIGSERIAL PRIMARY KEY,
    supplier_id INTEGER NOT NULL REFERENCES suppliers(id),
    report_month DATE NOT NULL,                   -- 报告月份（当月1日）
    production_units INTEGER DEFAULT 0,           -- 产品产量
    -- ↓ 原始能耗数据（供应商提交，可来自Excel或API）
    electricity_kwh REAL DEFAULT 0,               -- 外购电力
    natural_gas_m3 REAL DEFAULT 0,                -- 天然气
    steam_ton REAL DEFAULT 0,                     -- 蒸汽
    diesel_l REAL DEFAULT 0,                      -- 柴油
    gasoline_l REAL DEFAULT 0,                    -- 汽油
    -- ↓ 系统自动计算（供应商不填）
    calculated_co2_kg REAL DEFAULT 0,             -- 系统自动计算碳排
    carbon_score REAL DEFAULT 0,                  -- 碳评分（自动）
    carbon_level VARCHAR(10),                     -- 等级 A/B/C/D（自动）
    data_source VARCHAR(20) DEFAULT 'excel',      -- excel / api / sync
    submitted_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(supplier_id, report_month)
);
```

### 5.4 供应商上传记录表

```sql
CREATE TABLE supplier_upload_logs (
    id BIGSERIAL PRIMARY KEY,
    supplier_id INTEGER NOT NULL,
    file_name VARCHAR(200),
    records_imported INTEGER,
    status VARCHAR(20),  -- success / failed
    error_message TEXT,
    uploaded_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
