## 二、导入方式

### 方式一：API 接口导入（推荐用于批量/自动导入）

```
POST /api/import/{table_name}
Content-Type: multipart/form-data
参数：file = @your_data.xlsx
```

**响应格式：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "imported": 120,
    "table": "energy_records",
    "title": "能耗时序数据"
  }
}
```

### 方式二：通过系统管理页面导入（推荐日常使用）

菜单路径：**系统管理 → 数据导入**

页面提供了：
- 一键下载任意模板
- 上传确认导入
- 导入记录查看

## 三、导入模板说明

每个数据源都配有标准化 Excel 模板，可通过以下方式获取：

```
GET /api/import/{table_name}/template
```

### 3.1 核心业务数据模板

#### ① 耗能单元基础信息（device_groups）

> ⚠️ **关于粒度**：当前系统数据维度为 **生产单元级**（车间/产线）。系统支持三级粒度扩展：
> **生产单元**（当前）→ **设备组**（后续）→ **单个设备**（后续）
> 当前导入时请按**生产单元**填写数据，只需填名称和类型。后续阶段可逐级下钻。

| 字段 | 说明 | 类型 | 必填 | 示例 |
|------|------|------|------|------|
| 单元名称 | 耗能单元名称（当前=生产单元名） | 文本 | ✅ | A车间 |
| 单元类型 | production_unit / device_group / device | 选项 | ✅ | production_unit |
| 上级单元名称 | 最顶层留空，子级填父级名称 | 文本 | 选填 | |
| 额定功率(kW) | 单元装机总容量 | 数值 | 选填 | 500.0 |
| 位置 | 所在位置 | 文本 | 选填 | 1号厂房 |
| 描述 | 备注 | 文本 | 选填 | 空压车间 |

**第一阶段示例（按生产单元导入）：**

```
单元名称     单元类型          上级单元名称  额定功率  位置    描述
A车间       production_unit                 500.0    1号厂房  空压车间
B车间       production_unit                 800.0    2号厂房  机加工车间
C车间       production_unit                 300.0    3号厂房  组装测试
```

**后续阶段示例（设备组下钻）：**
```
A车间-空压机组  device_group    A车间         132.0    1号厂房A区
A车间-冷却系统  device_group    A车间          75.0    1号厂房B区

**后续阶段示例（单个设备）：**
```

## 四、导入步骤操作指南

### 4.1 通过 API 导入（开发人员/自动化）

```bash
# 1️⃣ 下载模板
curl -o template.xlsx "https://your-server/api/import/device_groups/template"

# 2️⃣ 按模板填写数据后导入
curl -X POST -F "file=@template.xlsx" "https://your-server/api/import/device_groups"

# 批量导入示例
for table in device_groups production_data alarm_rules suppliers; do
  curl -X POST -F "file=@${table}.xlsx" "https://your-server/api/import/${table}"
done
```

### 4.2 建议导入顺序

为确保数据完整性，建议按以下顺序导入：

```
Step 1: 耗能单元基础信息 (device_groups)            ← 先建层级
Step 2: 组织架构 (org_structure)
Step 3: 碳排放因子 (emission_factors)
Step 4: 碳配额 (carbon_quotas)
Step 5: 报警规则 (alarm_rules)
─────────────────────────────────
Step 6: 日产量数据 (production_data)
Step 7: 能耗时序数据 (energy_records)   ← 大批量数据
Step 8: 设备能耗数据 (device_energy)     ← 大批量数据
─────────────────────────────────
Step 9: 供应商碳数据 (suppliers)
Step 10: 数据源配置 (data_sources)                  ← 新增
Step 11: 碳交易价格趋势 (carbon_trade_prices)
Step 12: CCER项目 (ccer_projects)
Step 13: 能流分析配置 (energy_flow_config)
Step 14: 优化任务 (optimization_tasks)
Step 15: 导入记录 (import_logs)                     ← 新增
```

### 4.3 API 参考

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/import/tables` | 获取所有可导入表列表 |
| GET | `/api/import/{table_name}/template` | 下载指定表的空白模板 |
| POST | `/api/import/{table_name}` | 上传 Excel 导入数据 |
| GET | `/api/...` | 原有 API 接口不变 |

## 六、注意事项

1. **模板不变更**：请勿修改模板的表头行和说明行，仅从数据行（第5行起）填入数据
2. **时序数据批量**：能耗时序数据单次可导入10万+条，建议按天/周分批导入
3. **追加写入**：核心业务数据导入为追加模式（非覆盖）
4. **配置数据替换**：供应商/CCER等硬编码数据导入后，会替换全部内容（非追加）
5. **文件大小限制**：单次导入文件不超过 50MB
6. **原API接口不变**：所有原 API 接口继续可用，导入数据后自动生效
7. **数据验证**：系统会对导入数据做自动校验（必填字段、数值格式），不合规数据将被跳过并记录错误
