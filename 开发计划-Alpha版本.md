## 一、技术栈与技能要求

| 层 | 技术 | 技能要求 | 水平 |
|----|------|----------|------|
| **后端** | Python 3.14+ / FastAPI | RESTful API 设计、JWT 认证、中间件 | 熟练 |
| **数据库** | SQLite（开发）/ PostgreSQL + TimescaleDB（生产） | 数据库设计、SQL 优化、时序数据 | 熟练 |
| **Excel** | openpyxl | Excel 读写、模板生成、数据校验 | 熟悉 |
| **前端** | Vue 3 + Vite + ECharts + Pinia | 组件开发、路由守卫、状态管理、图表 | 熟练 |
| **部署** | Nginx + Supervisor | 反向代理、进程管理 | 熟悉 |
| **工具** | Git | 版本管理 | 熟练 |

## 四、开发顺序建议

按**依赖关系**排列，每一行可并行开发：

```
第1批（架构基础）：
  后端拆分 + 异常处理 + JWT + 数据库配置
  
第2批（可并行）：
  ├─ 数据字典（API + 页面）
  ├─ 数据源管理（API + 页面）
  ├─ 大屏 API + 心跳
  └─ 供应商管理（API + 页面）

第3批（依赖第2批）：
  供应商数据看板 + 上传接口
  
第4批（整合）：
  27 个页面数据切换 + 测试
```
