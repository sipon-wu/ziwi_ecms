## 二、问题详述与修正方案

### 🔴 P0 — 必须修正，影响系统健壮性

#### 问题1：数据库用 SQLite 先行，PostgreSQL 后续迁移

| 原文 | 问题 |
|------|------|
| "第一阶段继续使用SQLite（做WAL优化），提供PostgreSQL迁移脚本" | ⚠️ 生产系统第一天就不应该用SQLite |
| "PostgreSQL生产部署 → 第一阶段末" | ⚠️ 数据库是架构基础，必须在第一天确定 |

**修正方案**：
- 数据库设计**直接按 PostgreSQL + TimescaleDB** 写，不再留 SQLite 退路
- 开发调试用 SQLite（通过配置切换），生产部署必须 PostgreSQL
- 在 `backend/config.py` 中提供数据库连接配置切换：

```python
# config.py
import os
DB_TYPE = os.getenv("DB_TYPE", "sqlite")  # sqlite / postgres
SQLITE_PATH = os.getenv("SQLITE_PATH", "energy_data.db")
PG_DSN = os.getenv("PG_DSN", "postgresql://user:pass@localhost:5432/ecms")
```

- 后端所有 SQL 查询通过统一的 **repository 层** 调用，不直接写 raw SQL
- 第一阶段如果客户环境没有 PostgreSQL，可以用 SQLite，但**系统架构必须为 PostgreSQL 设计**

---

#### 问题2：数据源设计"Excel only，其他灰化"

| 原文 | 问题 |
|------|------|
| "MQTT采集源配置 → 界面预留，灰化不可用" | ⚠️ 把第一阶段临时限制写进了系统架构 |

**修正方案**：
- 后端实现统一的数据采集抽象层：

```python
# backend/collectors/base.py
from abc import ABC, abstractmethod

class DataCollector(ABC):
    @abstractmethod
    def fetch(self, source_config: dict, date: str) -> list:
        """从数据源获取指定日期的能耗数据"""
        ...

class ExcelCollector(DataCollector): ...    # 第一阶段
class MqttCollector(DataCollector): ...     # 第二阶段
class HttpCollector(DataCollector): ...     # 第二阶段
class ErpCollector(DataCollector): ...      # 第二阶段
```

- 前端数据源管理页面**直接展示所有类型**，Excel 有完整操作界面，其他类型显示"即将支持"状态提示（而不是灰化）
- 代码层面所有采集器平等对待，只通过配置决定启用哪些

---

#### 问题3：通知配置"界面预留，灰化不可用"

| 原文 | 问题 |
|------|------|
| "邮件服务器配置 → 界面预留，灰化不可用" | ⚠️ 同样是把临时限制写进架构 |

**修正方案**：
- 后端先实现通知服务抽象层（`notifiers/base.py`）
- 前端通知配置页面正常展示所有配置项，填写后后端先存储配置（即使不发信）
- 第一阶段缺省不启用发送，但配置可以保存
- 第二阶段启用时，配置直接在界面上启用即可

## 三、需修正的章节汇总

### 3.1 规格书第一章：系统概述

**需要改**：
```
❌ PostgreSQL + TimescaleDB (SQLite过渡，第一阶段保持SQLite可运行)
✅ PostgreSQL + TimescaleDB（生产必需配置）
   SQLite（仅开发调试用，通过 config.py 切换）
```

### 3.2 规格书第一章：核心约定

**需要改**：
```
❌ 数据库→第一阶段继续使用SQLite
✅ 数据库→按PostgreSQL设计，SQLite仅开发用
```

### 3.3 规格书第四章：配置管理

**需要改**：
```
❌ MQTT灰化、通知灰化
✅ 后端先实现抽象层，前端按类型展示，未就绪的标"即将支持"
```

### 3.4 规格书第七章：数据源方案

**需要重写**：
```
❌ 只讲Excel导入，只考虑第一阶段
✅ 统一DataCollector接口 + Excel/MQTT/HTTP Provider
   第一阶段只启用Excel Provider
```

### 3.5 规格书第八章：数据库设计

**需要重写**：
```
❌ 以SQLite为主，留迁移脚本
✅ 以PostgreSQL + TimescaleDB为主
   建表脚本使用PostgreSQL DDL
   SQLite仅通过兼容层提供开发调试
```
