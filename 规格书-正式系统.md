## 一、系统概述

### 1.1 系统定位
面向工业企业（特别是机械制造、化工、电子等行业）的 **能源消耗监测 + 碳排放管理一体化平台**，基于 AI 分析技术提供能效诊断、节能优化建议和碳资产管理能力。

### 1.2 目标用户
| 角色 | 使用场景 |
|------|----------|
| 企业能源管理员 | 日常能耗监测、报警处置、能耗报表 |
| 企业碳排放专员 | 碳排放核算、碳配额管理、碳交易、碳核查 |
| 企业管理者 | 驾驶舱总览、KPI监控、优化决策 |
| 系统管理员 | 用户管理、权限配置、系统参数维护 |

### 1.3 技术架构
```
┌─────────────────────────────────────────────────┐
│                 前端 (Vue3 + Vite)                │
│  ECharts 图表库  ·  Pinia 状态管理  ·  Axios API   │
├─────────────────────────────────────────────────┤
│              Nginx 反向代理 / 负载均衡              │
├─────────────────────────────────────────────────┤
│           后端 (FastAPI) — RESTful API              │
│  认证 · 数据采集 · 业务服务 · 报表生成 · AI分析     │
├──────────┬──────────────┬──────────────────────┤
│  PostgreSQL  │  TimescaleDB  │   Redis 缓存      │
│ (业务数据)  │  (时序数据)    │  (会话/队列)       │
└──────────┴──────────────┴──────────────────────┘
        ↑              ↑
   MQTT/HTTP     Excel/API
  (设备采集)     (ERP/地磅对接)
```

## 二、系统架构

### 2.1 前端
- **框架**：Vue 3 + Vite 构建
- **UI**：侧边栏导航（200px / 收缩 50px），深色侧边栏（#212529），内容区白色卡片
- **图表**：ECharts（柱状图/饼图/折线图/桑基图/仪表盘）
- **状态管理**：Pinia（store/），API 统一通过 `store.fetchJSONRaw(path)` 调用
- **路由**：Hash 路由（`createWebHashHistory`），共 27 个页面路由
- **构建**：`npm run build`，生产环境 base 路径 `/demo/ecms/`

### 2.2 后端
- **框架**：FastAPI（Python 3.14+）
- **端口**：8088
- **API 格式**：统一返回 `{"code": 0, "message": "success", "data": {...}}`
- **CORS**：全开放（`allow_origins=["*"]`）
- **当前数据库**：SQLite（`energy_data.db`）
- **数据模拟**：`energy_simulator.py` 按 5 分钟间隔生成 7 天模拟数据

### 2.3 数据流
```
[现场仪表/PLC] ──(MQTT)──→ [采集网关] ──(HTTP API)──→ [TimescaleDB]
                                                      ↓
[ERP系统] ──(API对接)──→ [业务数据] ──→ [PostgreSQL]
                                                      ↓
                                                  [FastAPI]
                                                      ↓
                                               [前端展示 / 报表导出]
```

### 3.1 驾驶舱（首页）

**页面路径**：`/dashboard`  
**页面文件**：`Dashboard.vue`

#### 功能点

| 功能 | 说明 | 后端 API |

- 同比变化百分比（今日 vs 昨日、本月 vs 上月、本年 vs 去年）

### 3.4 能耗分析

**包含页面**：
- `/analysis/consumption` — 消费量及强度
- `/analysis/energy_flow` — 能流分析
- `/analysis/benchmark` — 能效对标
- `/analysis/balance` — 能效平衡与优化

#### 3.4.1 消费量及强度
- KPI：总电耗(kWh)、综合能耗(吨标煤)、单位产品能耗(kWh/件)、单位产值能耗(kWh/万元)
- 日用电趋势（近7日折线图）
- 后端 API：`GET /api/analysis/energy_intensity`

#### 3.4.2 能流分析（桑基图）
- 从购入电力/天然气 → 设备组（空压机/机加工/测试台/辅助）→ 有效利用与损失
- 能流节点：购入电力、购入天然气、空压机组、机加工设备、测试台、辅助及照明、有效输出、损失
- 后端 API：`GET /api/analysis/energy_flow`

#### 3.4.3 能效对标
- 基于 GB 19153 标准，评价设备组能效等级（一级/二级/待提升）
- 指标：额定功率(kW)、实际平均功率(kW)、负载率(%)
- 后端 API：`GET /api/analysis/benchmark`

#### 3.4.4 能效平衡与优化
- 各设备组能耗分布饼图
- AI 优化建议列表（硬编码，正式系统需实时计算）
- 后端 API：`GET /api/analysis/balance`

## 六、数据库设计

### 6.1 生产数据库选型

| 选型 | 说明 |
|------|------|
| **PostgreSQL 15+** | 主数据库，存储业务数据和关系模型 |
| **TimescaleDB 扩展** | 时序数据超表，按时间分区自动管理 |
| **Redis 7+** | 缓存（会话 Token / 实时数据缓存 / 任务队列） |

### 6.2 现有表升级（11 表 → 13 表+）

#### 表 1：device_groups（设备组）
| 字段 | 类型 | 说明 | 升级变更 |
|------|------|------|---------|
| id | SERIAL PK | 主键 | 不变 |
| name | VARCHAR(100) | 设备组名称 | 不变 |
| type | VARCHAR(50) | 设备类型（compressor/milling/cnc/...） | 不变 |
| rated_kw | REAL | 额定功率(kW) | 不变 |
| ratio_pct | REAL | 占比 | 不变 |
| **meter_ids** | **JSON** | **关联采集表ID列表** | **新增** |
| **description** | **TEXT** | **设备组描述** | **新增** |

## 八、部署与运维方案

### 8.1 生产部署架构
```
[用户浏览器]
    ↓ HTTPS
[Nginx] ── SSL 证书 (Let's Encrypt)
    ├── 静态资源 → /var/www/ecms/
    └── API 代理 → FastAPI (127.0.0.1:8088)
                     ↓
              [Supervisor 管理]
                     ↓
          PostgreSQL + TimescaleDB
```

### 8.2 推荐配置

| 组件 | 推荐 |
|------|------|
| 服务器 | 4核8G 起，SSD 100G+ |
| 操作系统 | Ubuntu 22.04 LTS / CentOS 8+ |
| 数据库 | PostgreSQL 15 + TimescaleDB 2.x |
| 缓存 | Redis 7 (可选) |
| 前端 | Nginx 1.24+ |
| 后端 | Supervisor + FastAPI (uvicorn) |
| 容器化 | Docker Compose (推荐用于快速部署) |

### 8.3 运维方案

| 事项 | 方案 |
|------|------|
| 数据库备份 | 每日全量备份 + WAL 归档，保留30天 |
| 时序数据归档 | 按月分区，超1年数据迁移至冷存储 |
| 日志监控 | 后端日志 → 按天切割，保留90天 |
| 性能监控 | 系统资源监控（CPU/内存/磁盘），数据库慢查询日志 |
| 告警通知 | nginx 错误日志 + 后端异常 → 企业微信通知 |
| 版本发布 | 蓝绿部署或滚动更新 |

### 8.4 启动/停止命令

```bash
# 后端
supervisorctl start ecms-api
supervisorctl stop ecms-api

# 数据库
systemctl start postgresql
systemctl stop postgresql

# Nginx
nginx -s reload
nginx -s stop
```
