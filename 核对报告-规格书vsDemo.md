## 二、核心缺失：配置管理功能

### 2.1 当前参数配置只有 7 项

```
peak_start      → 峰时开始 08:00
peak_end        → 峰时结束 20:00
emission_factor → 电网排放因子 0.5566
carbon_budget_monthly → 月度碳预算 180
refresh_interval → 实时刷新间隔 10s
page_size       → 默认分页大小 20
alarm_retention_days → 告警保留天数 90
```

**这些只是系统级参数，缺乏业务级的配置管理。**

### 2.2 需要新增的配置管理功能

| 配置类别 | 当前状况 | 正式系统需要 | 对应页面影响 |
|----------|----------|-------------|-------------|
| **能源类型管理** | 硬编码（电力/天然气/蒸汽） | 用户自定义能源种类、单位、排放因子 | 碳核算、能流分析 |
| **设备类型管理** | DB 硬编码5类（compressor/milling/cnc...） | 用户增删改设备分类、额定功率 | 设备监控、能效分析 |
| **排放源管理** | 硬编码（外购电力/天然气/蒸汽） | 用户自定义排放源、范围(1/2/3)、因子 | 碳排放核算 |
| **产品目录** | 硬编码（DV-15） | 用户管理产品型号、BOM | 产品碳足迹 |
| **供应商管理** | 硬编码4个 | 用户增删改供应商信息 | 供应链碳管理 |
| **峰谷时段配置** | DB 里2个配置项 | 支持多时段、阶梯电价 | 用能结构、成本分析 |
| **数据采集源配置** | 🔴 完全缺失 | MQTT/HTTP/Excel/ERP 采集源管理 | 实时数据、导入 |
| **报警通知渠道** | 🔴 完全缺失 | 邮件/企微/webhook | 报警管理 |

### 2.3 用户说的"信息源及其类别更多的配置管理功能"具体指什么？

从产品角度理解，用户需要的是**可配置的数据字典能力**：

```
系统配置（当前7项，需扩展）
├─ 基础参数
│  ├─ 排放因子 → 可配置多个因子、不同年份、不同来源
│  ├─ 峰谷时段 → 可配置多个时段，分别设置电价
│  └─ 碳预算 → 月度/季度/年度可调
│
├─ 数据字典（🔴 缺失，最需要新增）
│  ├─ 设备类型管理 → 新增/编辑/删除设备分类
│  ├─ 能源类型管理 → 电力/天然气/蒸汽/水/压缩空气
│  ├─ 排放源类别 → 范围1/2/3的自定义排放源
│  ├─ 产品型号管理 → 产品目录维护
│  └─ 供应商管理 → 供应链碳管理的源数据
│
├─ 数据源配置（🔴 缺失）
│  ├─ 设备采集源 → MQTT Broker 地址/端口/主题
│  ├─ HTTP 上报 → API Key/白名单
│  ├─ ERP 对接 → 接口地址/认证方式
│  └─ 数据导入记录 → 已导入的文件清单
│
└─ 通知配置（🔴 缺失）
   ├─ 邮件服务器 → SMTP 配置
   ├─ 企业微信 → Webhook URL
   └─ 告警升级规则 → 超时自动升级
```

## 三、综合建议

### 维持 Demo 前端样式，新增配置管理页面

按用户要求"就按 demo 前端进行正式系统开发"，UI 风格完全保留。在此基础上新增：

| 页面 | 路径建议 | 说明 |
|------|----------|------|
| 数据字典管理 | `/system/dict` | 设备类型/能源类型/排放源类别/产品型号/供应商的统一管理 |
| 数据源管理 | `/system/datasource` | 采集源、对接、导入记录管理 |
| 通知配置 | `/system/notify` | 邮件/企微/告警升级规则 |

### 页面原型建议

**数据字典管理页面**（类似当前参数配置的增强版）：

```
┌─────────────────────────────────────────────────┐
│  数据字典管理                                      │
├─────────────────────────────────────────────────┤
│  [设备类型] [能源类型] [排放源] [产品型号] [供应商]   │  ← Tab 切换
├─────────────────────────────────────────────────┤

│  │ 设备类型列表            │  │ 编辑             ││

│  │  testbench  测试台     │  │ 位置: A车间      ││

└─────────────────────────────────────────────────┘
```

### 数据库扩展

```sql
-- 数据字典主表（替代硬编码的设备类型/能源类型等）
CREATE TABLE data_dict_types (
    id SERIAL PRIMARY KEY,
    dict_code VARCHAR(50) UNIQUE NOT NULL,  -- device_type / energy_type / emission_source
    dict_name VARCHAR(100) NOT NULL,
    description TEXT
);

-- 字典项
CREATE TABLE data_dict_items (
    id SERIAL PRIMARY KEY,
    dict_type_id INTEGER REFERENCES data_dict_types(id),
    item_key VARCHAR(50) NOT NULL,     -- compressor / electricity
    item_value VARCHAR(100) NOT NULL,  -- 空压机组 / 电力
    sort_order INTEGER DEFAULT 0,
    extra JSON,                         -- 扩展字段（如额定功率、排放因子等）
    enabled INTEGER DEFAULT 1
);

-- 数据源配置
CREATE TABLE data_sources (
    id SERIAL PRIMARY KEY,
    source_name VARCHAR(100) NOT NULL,   -- 电表1号 / ERP系统
    source_type VARCHAR(50) NOT NULL,    -- mqtt / http / erp / excel
    config JSON,                          -- { host, port, topic, ... }
    status VARCHAR(20) DEFAULT 'offline',
    last_collect_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
