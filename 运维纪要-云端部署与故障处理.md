第一篇：Alpha 版本初次部署（2026-06-04）
================================================================================

一、部署概况
  版本：Alpha 1.0
  后端：Python FastAPI → uvicorn → port 8088
  前端：Vue 3 + Vite → dist/ → /var/www/demo/ecms/
  数据库：SQLite → /var/www/demo/ecms-backend/energy_data.db
  Nginx：Let's Encrypt SSL → HTTPS 443

二、部署步骤
  1. 前端打包：vite build → dist/（含 assets/ + index.html + ziwilogo.png）
  2. 上传前端到 /var/www/demo/ecms/
  3. 上传后端代码到 /var/www/demo/ecms-backend/
  4. pip3 install fastapi uvicorn
  5. nohup uvicorn main:app --host 127.0.0.1 --port 8088 &
  6. 配置 Nginx location /demo/ecms/
  7. 配置 Nginx location /demo/ecms/api/
  8. nginx -t && nginx -s reload
  9. 配置 SSL 证书（Let's Encrypt）

三、Nginx 配置（初始版）
    location ^~ /demo/ecms/ {
        root /var/www;
        index index.html;
        try_files $uri $uri/ /demo/ecms/index.html;
    }
    location ^~ /demo/ecms/api/ {
        proxy_pass http://127.0.0.1:8088/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

四、故障处理：Logo 不显示
  现象：页面加载正常，但顶部 Logo 图标 404
  排查：
    - 前端引用路径：<img src="/ziwilogo.png">
    - Nginx root 路径为 /var/www，请求 /ziwilogo.png 指向 /var/www/ziwilogo.png，不存在
    - 实际文件在 /var/www/demo/ecms/ziwilogo.png
  修复：
    location = /ziwilogo.png {
        alias /var/www/demo/ecms/ziwilogo.png;
    }

五、遗留问题
  □ 前端 baseURL 未统一加 /demo/ecms/ 前缀，API 请求路径为裸 /api/xxx
  □ 因上述原因，需为每个 /api/xxx/ 模块单独写 Nginx 代理规则（共8条）
  □ 部署无 systemd/supervisor 守护进程，后端进程需手动管理


第二篇：Beta 版本升级 + 登录故障处理（2026-06-10）
================================================================================

一、升级概况
  版本：Beta 20260609
  新特性：
    - 15张业务表，11,877行数据
    - 独立表 daily_energy（10种能源类型）
    - simulate_monthly.py v2 双分支模拟引擎
    - 47项 API 全量测试通过
    - 登录页 guest/123 自动填充
    - 每日 00:05 数据自动重置（reset_demo.py）

二、故障现象
  访问 https://www.ziwi.cn/demo/ecms/#/login，点击登录后报错：
  "Unexpected token '<', '<!doctype...' is not valid JSON"
  前端期望 JSON，实际收到 HTML。

三、排查过程（云端助手执行）
  步骤  排查内容          结论
  ─────────────────────────────────────────────────────────
  1     Nginx 错误日志     无相关报错
  2     curl /api/auth/login  后端返回正确 JSON ✅
  3     后端进程           uvicorn 运行正常，8088 正常 ✅
  4     后端路由           登录接口路径 /api/auth/login ✅
  5     前端 JS 调用路径    /api/auth/login ✅
  6     Nginx 配置         发现根本原因（见下）

四、根本原因
  Nginx 配置中存在路由优先级冲突：

    # ✅ demo 的 API 代理
    location ^~ /demo/ecms/api/ {
        proxy_pass http://127.0.0.1:8088/api/;
    }
    # ❌ 主站 /api/ 代理拦截了 /api/auth/login，转发到 5000 端口
    location /api/ {
        proxy_pass http://127.0.0.1:5000/api/;
    }

  前端页面托管在 https://www.ziwi.cn/demo/ecms/，但发出的 API 请求
  路径是 /api/auth/login（不带 /demo/ecms/ 前缀），
  因此被底部主站规则拦截，转发到 5000 端口 → 返回 HTML 页面。

五、解决方案
  在 Nginx 配置中，于主站 location /api/ 之前补充：

    location ^~ /api/auth/ {
        proxy_pass http://127.0.0.1:8088/api/auth/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

  执行 nginx -t && nginx -s reload 后登录恢复正常。

六、补充修复
  1. 后端部署时缺少 openpyxl（Excel 模板引擎）
     执行 pip3 install openpyxl fastapi uvicorn bcrypt pyjwt pydantic
  2. 旧进程占用 8088，新后端无法启动
     执行 fuser -k 8088/tcp 强杀后重启
  3. 数据库从旧版 1.5MB 升级到新版 3.3MB
     备份旧 DB → 上传新 DB → 重启后端

七、验证结果
  ✅ curl 直接请求 /api/auth/login 返回正确 JSON + token
  ✅ Nginx 透传 /api/auth/login 返回正确 JSON + token
  ✅ https://ziwi.cn/demo/ecms/ 登录页一次点击登录成功
  ✅ 全链路：浏览器 → Nginx → Backend(8088) → SQLite


第三篇：运维 SOP & 后续建议
================================================================================

一、常用命令
  # Nginx
  nginx -t                    # 语法检查
  nginx -s reload             # 重载配置

  # 后端
  cd /var/www/demo/ecms-backend
  nohup python3 -m uvicorn main:app --host 127.0.0.1 --port 8088 > backend.log 2>&1 &
  pkill -f "uvicorn main:app" # 停止
  fuser -k 8088/tcp           # 强杀端口

  # 查看状态
  tail -f backend.log         # 实时日志
  curl http://localhost:8088/api/heartbeat  # 健康检查

二、文件路径
  前端静态文件  /var/www/demo/ecms/
  后端代码      /var/www/demo/ecms-backend/
  数据库        /var/www/demo/ecms-backend/energy_data.db
  Nginx 配置    /etc/nginx/sites-enabled/ziwi.cn
  SSL 证书      /etc/letsencrypt/live/ziwi.cn/

三、后续建议
  1. 前端 baseURL 统一加前缀 /demo/ecms/api/
     → 与 Nginx 的 location ^~ /demo/ecms/api/ 规则对应
     → 彻底避免与主站路由冲突
  2. 补全 /api/xxx/ 各路由的 Nginx 代理规则
      目前 /api/auth/、/api/dashboard/、/api/carbon/ 等需要独立配置
  3. 设置 systemd 或 supervisor 守护后端进程
     防止服务器重启后后端不自动启动
  4. 备份文件不要放在 /etc/nginx/sites-enabled/
     → Nginx 会加载该目录下所有文件
     → 备份应放到 /tmp/ 或其他目录

四、定时任务 (Cron) — 2026-06-11 更新
  服务器 crontab (crontab -l):
    0 0 * * * cd /var/www/demo/ecms-backend && python3 daily_refresh.py >> daily_refresh.log 2>&1
    0 0 1 * * cd /var/www/demo/ecms-backend && python3 monthly_guest_cleanup.py >> monthly_cleanup.log 2>&1

  数据策略 (详见 daily_refresh.py / monthly_guest_cleanup.py):
    每日 0:00 — daily_refresh.py
      ① 补充今日 energy_records 288 点（如已有则跳过）
      ② 补充今日 device_energy 设备能耗（如已有则跳过）
      ③ 不触碰配置表和历史数据
    每月1日 0:00 — monthly_guest_cleanup.py
      ① 清除 guest 操作记录（DELETE FROM operation_logs WHERE username='guest'）
      ② 回归干净的演示状态
      ③ admin 数据不受影响
